## Core Claims

- (§1 Introduction) "In emerging agentic systems, code is no longer only a target output. It increasingly serves as an operational substrate for agent reasoning, acting, environment modeling, and execution-based verification."
- (§1) "We frame this shift through the lens of *agent harnesses* and introduce *code as agent harness*: a unified view that centers code as the basis for agent infrastructure." — harness 개념을 survey 차원에서 정식화하고 code 를 그 substrate 로 통합.

## Key Concepts & Definitions

- **Agent Harness (정식 정의, §1)**: "the software layer that surrounds an LLM with tools, APIs, sandboxes, memory, validators, permission boundaries, execution loops, and feedback channels, thereby turning a stateless model into a functional agent capable of long-running task execution." — 블로그발 "harness" 를 학술 정의로 고정한 문장.
- **Three coupled elements (§1)**:
  1. *model-internal capabilities* — reasoning, perception, planning, simulation, evaluation
  2. *system-provided harness infrastructure* — predefined tools, APIs, sandboxes, validators, permission systems
  3. *agent-initiated code artifacts* — agent 가 생성·실행·관찰·수정·persist·공유하는 interactive code object
- **Code 의 scope (§2)**: "executable or machine-checkable artifacts" — program, script, formal spec, proof script, API schema, tool definition, test, repository, simulator, config, 그리고 trace·log 같은 code-adjacent execution artifact 포함.
- **3-layer 구성**: Harness Interface(§2, code-for-reasoning / acting / environment) · Harness Mechanisms(§3, planning·memory·tool use·control loop·harness optimization) · Scaling the Harness(§4, multi-agent orchestration over shared code).

## Patterns Covered

- Program-aided reasoning (PAL, PoT, Chain of Code)
- Formal verification interface (Lean, SAT/SMT solver)
- Skill library / affordance-based action (SayCan, BOSS)
- Programmatic policy generation (CaP, RoboCodeX, Code-BT)
- Lifelong skill accumulation (Voyager, LYRA, SkillsCrafter)
- World modeling via execution trace (WorldCoder, CWM)
- Repository-level evaluation (SWE-bench, AgentBench, InterCode)
- Memory layering (working / semantic / experiential / long-term)
- Multi-agent coordination over shared harness (AutoGen, MetaGPT)
- **Plan-Execute-Verify** feedback loop

## Generation Mapping

- **Harness Engineering 세대의 정식 학술 정의 출처**. 매뉴얼이 Anthropic/Addy 블로그로 명명하는 "harness" 를 survey 가 한 문장 정의로 고정 — "turns a stateless model into a functional agent". 세대사 1부의 harness 정의 인용 시 tier-4 backing 으로 가장 직접적.
- 매뉴얼 실무 패턴 대응: **maker/verifier**(Plan-Execute-Verify, execution-based verification), **상태 파일 영속성**(stateful medium, working→long-term memory layering), **subagent 분업**(§4 multi-agent orchestration, role specialization, workflow topology), **eval/golden set 회귀**(§5.2 "regression-free harness improvement" open challenge — 매뉴얼의 골든셋 회귀 주장과 직결).
- "Science of harness engineering" 부재를 open challenge(§5.2)로 명시 — 매뉴얼이 주장하는 '엔지니어링 분과로서의 harness' 가 아직 미정립임을 학술적으로 인정.

## Method & Evidence

- 형식: comprehensive survey — **200+ representative works (through 2026)** 를 taxonomy 로 종합.
- 근거 유형: system case study(Claude Code, Codex, LangChain, 엔터프라이즈 플랫폼), benchmark 결과, 산업 구현 패턴(Anthropic long-running harness, Cursor autonomous coding), 알고리즘 비교(AutoCodeRover retrieval 등).
- 인용 벤치(§2.3.3): SWE-bench(repo-level unit-test), CRUXEval(I/O prediction), LiveCodeBench(continuously updated pipeline), AgentBench(multi-environment). — 정량 수치 표는 survey 특성상 자체 측정 없음, 외부 벤치 참조.

## Limitations

- Survey — 자체 실험·정량 측정 없음, 외부 벤치 참조에 의존.
- 저자 스스로 7대 open challenge 명시(§5.2): (1) final task success 너머의 평가 지표 부재 (2) incomplete feedback 하 semantic verification (3) regression-free harness improvement (4) shared state 의 transactional 일관성 (5) safety-critical action 의 human oversight (6) multimodal harness (7) **science of harness engineering 부재(principled design methodology 결여)**.
- "Planning quality improvements depend heavily on surrounding execution conditions" — planning 개선 효과가 환경·feedback·budget 에 강하게 의존, 일반화 주의.
