# Adapters

Adapters map the model-agnostic core contract in `../core/CORE.md` onto a concrete agent runtime. The adaptation boundary is defined in `../core/ADAPTATION.md`; the current surface-by-surface migration map is `../core/ADAPTATION_INVENTORY.md`.

An adapter owns runtime-specific details:

- bootstrap file names and loading behavior;
- hook/event schemas;
- slash command registration;
- permission model and tool names;
- status UI;
- compatibility shims for local paths and legacy names.
- runtime-home projection from the neutral repo into the tool's config directory.

The core owns workflow meaning, artifact layout, memory lifecycle, and safety invariants. New adapters should not redefine those concepts; they should document how their runtime implements them.

Path convention: portable docs use `<agent-home>`. Shell code should resolve it with `utilities/agent-home.sh` or the same rule: `AGENT_HOME`, then an adapter compatibility variable such as `CLAUDE_HOME`, then `$HOME/agent_setting` when present, then the adapter legacy default.

## Runtime Home Projection

The neutral repo should live outside vendor runtime homes:

```text
$HOME/agent_setting/
```

Runtime homes such as `$HOME/.claude`, `$HOME/.codex`, `$HOME/.config/opencode`, and `$HOME/.local/share/opencode` are owned by their tools and may contain credentials, session databases, logs, caches, and other mutable state. Do not make those runtime homes the canonical git repo. Instead, project the harness into them:

- symlink shared directories only when the adapter declares them as portable or compatibility passthrough;
- treat compatibility passthrough as migration debt, not proof that a shared file is portable;
- keep runtime-owned files local to the runtime home;
- keep adapter-owned bootstrap/config files either symlinked from `<agent-home>/adapters/<adapter>/...` or generated by an adapter install script.

Whole-directory symlinks (`$HOME/.claude -> $HOME/agent_setting`) are discouraged because they mix runtime state with the portable harness.
